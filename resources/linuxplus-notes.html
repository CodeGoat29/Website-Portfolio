CompTIA Linux+ (XK0-005) Study Notes

Structured notes aligned to the Linux+ exam domains

Table of Contents

1.0 System Management (32%) 2.0 Security (21%) 3.0 Scripting, Containers, and Automation (19%) 4.0 Troubleshooting (28%)

1.0 System Management (32%)

1.1 Summarize Linux fundamentals

  • Filesystem Hierarchy Standard (FHS): /boot, /proc, /sys, /var, /usr, /lib, /dev, /etc, /opt, /bin, /sbin, /home, /media, /mnt, /root, /tmp
  • Basic boot process: BIOS, UEFI, GRUB2, kernel panic
  • Boot commands: mkinitrd, grub2-install, grub2-mkconfig, grub2-update, dracut, initrd.img, vmlinuz
  • Boot sources: PXE, USB, ISO
  • Device types in /dev: Block devices, character devices, special character devices
  • Special devices: /dev/null, /dev/zero, /dev/urandom
  • Package compilation: ./configure, make, make install
  • Storage concepts: File, block, object storage; MBR, GPT; FUSE; RAID (striping, mirroring, parity)
  • Hardware info: lspci, lsusb, dmidecode

1.2 Manage files and directories

  • File editing: sed, awk, printf, nano, vi(m)
  • Compression/archiving/backup: gzip, bzip2, zip, tar, xz, cpio, dd
  • Metadata: stat, file
  • Links: soft and hard
  • Copying between systems: rsync, scp, nc
  • File/dir ops: mv, cp, mkdir, rmdir, ls, pwd, rm, cd, ., .., ~, tree, cat, touch

1.3 Configure and manage storage

  • Disk partitioning: fdisk, parted, partprobe
  • Disk usage: df, du
  • LVM: pvs, vgs, lvs, pvcreate, vgcreate, vgextend, lvcreate, lvresize, lvchange
  • Mounting: systemd.mount, /etc/fstab, mount
  • Encryption: LUKS
  • Filesystem tools: XFS, Ext4, Btrfs
  • RAID: mdadm, /proc/mdstat
  • SAN/NAS: multipathd, NFS, SMB/CIFS
  • Storage hardware: lsscsi, lsblk, blkid, fcstat

1.4 Processes and services

  • systemctl: stop, start, restart, status, enable, disable, mask
  • Signals: SIGTERM, SIGKILL, SIGHUP
  • Process tools: top, ps, lsof, htop
  • Scheduling: cron, crontab, at
  • Priorities: nice, renice
  • States: zombie, sleeping, running, stopped
  • Job control: bg, fg, jobs, Ctrl+Z, Ctrl+C, Ctrl+D
  • Process lookup: pgrep, pkill, pidof

1.5 Networking tools/config files

  • Interfaces: iproute2 (ip, ss), NetworkManager (nmcli), net-tools (ifconfig), ifcfg, hostname, arp, route
  • Config paths: /etc/sysconfig/network-scripts/
  • Name resolution: nsswitch, /etc/resolv.conf, systemd (hostnamectl, resolvectl), bind-utils (dig, nslookup, host), WHOIS
  • Monitoring: tcpdump, wireshark/tshark, netstat, traceroute, ping, mtr
  • Remote tools: SSH, cURL, wget, nc, rsync, SCP, SFTP

1.6 Build and install software

  • Package managers: DNF, YUM, APT, RPM, dpkg, ZYpp
  • Sandboxed apps: snapd, Flatpak, AppImage
  • Updates: kernel updates, package updates

1.7 Manage software configurations

  • Config procedures: restart service, reload service, .rpmnew, .rpmsave
  • Repo files: /etc/apt.conf, /etc/yum.conf, /etc/dnf/dnf.conf, /etc/yum.repo.d, /etc/apt/sources.list.d
  • Kernel params: sysctl, /etc/sysctl.conf
  • Kernel modules: lsmod, insmod, rmmod, modprobe, modinfo
  • Common services: SSH, NTP, syslog, chrony
  • Localization: timedatectl, localectl

2.0 Security (21%)

2.1 Security best practices

  • PKI: public/private key, self-signed cert, digital signature, wildcard cert, hashing, CAs
  • Cert use cases: SSL/TLS, certificate authentication, encryption
  • Authentication: tokens, MFA, PAM, SSSD, LDAP, SSO
  • Hardening: scanning, secure boot (UEFI), logging config, umask, disable insecure services, password strength, remove unused packages, kernel tuning, secure service accounts, host firewall

2.2 Identity management

  • Account tools: useradd, groupadd, userdel, groupdel, usermod, groupmod, id, who, w
  • Defaults/config: /etc/passwd, /etc/group, /etc/shadow, /etc/profile, /etc/skel, .bash_profile, .bashrc
  • Account management: passwd, chage, pam_tally2, faillock, /etc/login.defs

2.3 Firewalls

  • Use cases: open/close ports, check config, enable/disable IP forwarding
  • Technologies: firewalld, iptables, nftables, UFW
  • Features: zones, services, stateful, stateless

2.4 Remote connectivity

  • SSH configs: /etc/ssh/sshd_config, /etc/ssh/ssh_config, ~/.ssh/known_hosts, ~/.ssh/authorized_keys, ~/.ssh/config
  • SSH commands: ssh-keygen, ssh-copy-id, ssh-add
  • Tunneling: X11 forwarding, port forwarding, dynamic forwarding
  • Run as another user: /etc/sudoers, PolicyKit, sudo, visudo, su -, pkexec

2.5 Access controls

  • Permissions: ACL, SUID, SGID, sticky bit
  • SELinux: context permissions, labels, autorelabel, booleans, enforcing/permissive/disabled, targeted/minimum
  • AppArmor: application permissions
  • Utilities: chown, umask, chmod, getfacl, setfacl, ls, setenforce, getenforce, chattr, lsattr, chgrp, setsebool, getsebool, chcon, restorecon, semanage, audit2allow

3.0 Scripting, Containers, and Automation (19%)

3.1 Shell scripting

  • Loops: while, for, until
  • Conditionals: if, switch/case
  • Expansion: globbing, brace expansions
  • Comparisons: arithmetic, string, boolean
  • Variables and regex
  • Streams: |, ||, >, >>, <, <<, &, &&, stderr/stdout
  • Here docs, exit codes, built-ins: read, echo, source
  • Utilities: awk, sed, find, xargs, grep, egrep, tee, wc, cut, tr, head, tail
  • Env vars: $PATH, $SHELL, $?
  • Paths: relative and absolute

3.2 Containers

  • Management: start/stop, inspect, list, deploy images, connect, logging, expose ports
  • Images: build, push, pull, list, rmi

3.3 Git basics

  • clone, push, pull, commit, add, checkout, branch, tag, gitignore

3.4 IaC + CI/CD

  • Formats: YAML, JSON
  • Utilities: Ansible, Puppet, Chef, SaltStack, Terraform
  • CI/CD use cases
  • Advanced Git: merge, rebase, pull requests

3.5 Container, cloud, orchestration concepts

  • Kubernetes: benefits, pods, sidecars, ambassador containers
  • Single-node/multicontainer: Compose
  • Persistent storage, networks: overlay, bridging, NAT, host
  • Service mesh
  • Bootstrapping: cloud-init
  • Registries

4.0 Troubleshooting (28%)

4.1 Storage issues

  • High latency, I/O wait, low throughput, low IOPS
  • Capacity issues, inode exhaustion
  • Filesystem issues: corruption, mismatch
  • I/O scheduler, device issues (NVMe, SSD, trim, RAID, LVM, errors)
  • Mount option problems

4.2 Network issues

  • Subnet/routing, firewall issues, interface errors (drops/collisions/link)
  • Bandwidth limitations, latency
  • Name resolution issues (DNS)
  • Testing remote systems: Nmap, openssl s_client

4.3 CPU and memory issues

  • Runaway/zombie processes, high CPU, high load, run queues
  • CPU times: steal, user, system, idle, iowait
  • Priorities: nice, renice
  • Memory exhaustion, OOM, swapping
  • Hardware: lscpu, lsmem, /proc/cpuinfo, /proc/meminfo

4.4 User access and permissions

  • User login/file access issues, group/context/permissions, ACL/attributes
  • Password issues, privilege elevation, quota issues

4.5 Systemd troubleshooting

  • Unit files: services, ExecStart/ExecStop, before/after, type, user, requires/wants
  • Timers: OnCalendar, OnBootSec
  • Mounts: naming, what/where/type/options
  • Targets: default, multiuser, network-online, graphical
  • Common problems: name resolution failure, crashes, time-zone config, boot issues, journal issues, services not starting on time
